# 00 · 总览

## 一句话定义

**Deepulse** = 个人 AI 电台 PWA. 浏览器开 `localhost:3000`, **LLM 当 DJ 大脑**给你说话+选歌, **网易云** API 提供音源 + 元数据, **流式 TTS** 把 DJ 的串场词变成声音, 在浏览器里用 `<audio>` + Web Audio API 播放.

## 顶层数据流 (一次"用户输入 → DJ 回话 → 切歌"完整回合)

```mermaid
sequenceDiagram
    participant U as 用户 (浏览器)
    participant PWA as Next.js PWA (:3000)
    participant SRV as Fastify Server (:8787)
    participant BR as LLM Brain
    participant TTS as TTS Server (VoxCPM :8001 / SoVITS :8000 / mock)
    participant NCM as 网易云 API

    U->>PWA: 在 DjChat 输入 "放点周杰伦"
    PWA->>SRV: WS user_msg + DjContext (当前歌/队列长度)
    SRV->>BR: brain.stream(messages) (OpenAI 兼容 SSE)
    BR-->>SRV: token 流 "好,这就放<<play:周杰伦 稻香>>"
    SRV->>SRV: SentenceSegmenter 按句切
    par 异步合成每句
        SRV->>TTS: synthesize(句子) → audioUrl
    and 解析 inline action 标签
        SRV->>SRV: parseInlineActions → ParsedAction
    end
    SRV-->>PWA: WS token / sentence / audio / action / reply_done
    PWA->>PWA: SequentialAudioQueue 顺序播 DJ 配音 (ducking 压音乐 25%)
    PWA->>SRV: GET /api/search?q=周杰伦 稻香
    SRV->>NCM: cloudsearch(keywords)
    NCM-->>SRV: 歌曲列表
    SRV-->>PWA: { songs: [...], total }
    PWA->>SRV: GET /api/song/{id}/url
    SRV->>NCM: song_url_v1
    NCM-->>SRV: NCM CDN audio URL
    SRV-->>PWA: { url, quality }
    PWA->>U: <audio> 播放新歌, 顺手切到 Listen 模式
```

## 单用户应用的简化

这套架构有意识地**没做**多租户:

- 网易云只存一个 cookie (`ncm_account.id=1` 单行表)
- snapshot 只存一份 (`ncm_snapshot.id=1` 单行表)
- DJ 长期记忆只一个文件 (`dj-long-term.md`)
- Redis key 也没分 user (`deepulse:mem:active` / `deepulse:mem:session` 全局两个 key)

意识到这点对加新功能很重要: 别为想象中的"多用户"额外加 user_id 字段或前缀, 没有这个需求.
